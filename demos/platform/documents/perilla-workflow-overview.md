# Perilla 工作流编排概览

Perilla 是本地优先的 AI 推理与智能体平台。工作流（Workflow）将多个步骤编排为可复现的 DAG。

## 核心能力

- **人工审批（approval）**：在关键步骤暂停，等待用户确认后再继续执行。
- **Checkpoint**：对节点输出做结构化验收（required_keys），未通过可阻断或重试。
- **Fork / Join**：并行执行多路分支后汇聚结果，适合「多源调研」类任务。
- **Verify Loop**：迭代执行直到验收通过或达到最大轮次。

## 与纯 Chat 的差异

| 维度 | 纯对话 | 工作流 |
|------|--------|--------|
| 可复现 | 依赖单次提示 | 版本化 DAG，可发布 |
| 人机协同 | 弱 | 审批门、暂停/恢复 |
| 多 Agent | 需手动切换 | 节点级绑定 Agent / Skill |
