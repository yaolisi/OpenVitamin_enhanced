# Perilla 平台产品原则（2026-05）

本文档固化产品方向共识，与 [路线图评审附录](./ROADMAP_REVIEW_APPENDIX_ZH.md) 及实现对照使用。

---

## 1. 平台能力优先

- **发展重心**：推理网关、Agent Runtime、Workflow 控制面、多 Agent 协同、可观测与插件扩展。
- **行业化**：参考架构、合规工具包、连接器以 **Enterprise / 专版** 交付，不阻塞开源核心的简洁与可替换性。
- **安全治理**：**可接受且必要**，前提是不显著拖累推理热路径（见 §2）。

---

## 2. 安全治理与性能

| 原则 | 说明 |
|------|------|
| 热路径最小化 | TLS、鉴权、租户校验保留；KMS 轮询、DLP、外发审计走异步或旁路 |
| 默认关、按需开 | 专网单机可仅 API Key；企业版 Helm values 打开 OIDC/KMS |
| 可观测替代隐式魔法 | 审批、Checkpoint、correlation_id 显式可审计 |

---

## 3. 多 Agent 协同（向内增强）

**目标**：用户感知为「组队完成任务」，而非散落的多条 Chat。

| 层级 | 能力 |
|------|------|
| 会话级 | `correlation_id` / `orchestrator_agent_id` 写入 Agent 会话；运行页、执行页可查看与复制 |
| 编排级 | Workflow **Fork/Join**、多 **Agent** 节点、审批与验收门控 |
| 控制台 | 智能体列表协同提示；工作流运行 **填充协同上下文**；Demo 导入后 **预填示例入参** |

后端基线：`core/agent_runtime/collaboration.py`、`/api/collaboration/*`、Workflow `global_context` 归一化。

---

## 4. 易用性

- **主路径**：模型 → 对话 → 智能体 → 工作流（见 README / 新人 playbook）。
- **预检与 Demo**：导入演示项目后自动预检；运行跳转运行页并带 `sample_input`。
- **术语**：Fork vs Parallel 限流、审批 vs Checkpoint 在教程与预检文案中区分。

---

## 5. 信息架构（IA）

- **平台**（侧栏）：对话、智能体、工作流。
- **能力**：知识库、技能、模型。
- **工具**：文生图（不作为与对话并列的首屏能力）。
- **系统**：日志、设置。

文生图仍保留完整功能路由 `/images`，仅从「门面」降级为「工具」。
